# Internationalization (i18n)

The platform supports three languages for the user interface and article content: **Catalan** (`ca`), **Spanish** (`es`), and **English** (`en`). The user's language preference is stored in their profile and applied consistently across the app.

---

## Overview

- **UI text** (nav, buttons, labels, form fields, error messages, loading states) is translated via Flask-Babel and gettext catalogs.
- **Article content** is rewritten by the pipeline in the user's language; the feed and article views serve the correct language variant from `cluster_rewrites`.
- **Timestamps** (e.g. "5 minutes ago") use the `humanize` library with locale activation per request.
- **TTS labels** ("Listen", "Stop") are passed from the server to the inline JavaScript via `data-*` attributes on the body element.

---

## Locale Selection

| Context | Source | Fallback |
|---------|--------|----------|
| Authenticated user | `user_profiles.language` from DB | Config `rewriting.default_language` (`ca`) |
| Anonymous (login/register) | `Accept-Language` header | Config default (`ca`) |

Once logged in, the profile language overrides the browser preference. Anonymous users see login/register in their browser's preferred language when supported.

## Files and Structure

### Dependencies

- **Flask-Babel** — locale selection, `gettext`/`_()` in templates and Python, translation loading.

### Configuration

- **`babel.cfg`** — Tells `pybabel extract` which files to scan (Python `**/*.py`, Jinja2 `templates/**/*.html`).
- **`app/i18n_strings.py`** — Contains literal strings for config labels (Catalan, Spanish, English, Neutral, Simple) so they are extracted; do not import this module.

### Translation Catalogs

```
translations/
├── ca/LC_MESSAGES/messages.po   # Catalan
├── ca/LC_MESSAGES/messages.mo
├── es/LC_MESSAGES/messages.po   # Spanish
├── es/LC_MESSAGES/messages.mo
├── en/LC_MESSAGES/messages.po   # English (identity)
└── en/LC_MESSAGES/messages.mo
```

### App Integration (`app/__init__.py`)

- **Babel** — Initialized with `locale_selector=get_locale`.
- **`get_locale()`** — Returns user profile language when authenticated, otherwise `Accept-Language` best match.
- **Context processor** — Injects `profile`, `locale`, `is_admin`, `current_user_email` into all templates.
- **`activate_humanize_locale()`** — `@before_request` hook that calls `humanize.activate(locale)` for relative time formatting.

## Template Usage

Wrap user-facing strings in templates:

```html
{{ _('Log in') }}
{{ _("Today's feed") }}
{{ ngettext('%(num)d article', '%(num)d articles', count)|format(num=count) }}
```

Language dropdown options come from config and are translated:

```html
{% for lang in languages %}
<option value="{{ lang.id }}">{{ _(lang.label) }}</option>
{% endfor %}
```

The `<html>` tag uses the resolved locale:

```html
<html lang="{{ locale or 'en' }}">
```

---

## Python Usage

Use `gettext` for user-facing strings in routes:

```python
from flask_babel import gettext

return render_template("login.html", error=gettext("Invalid email or password."))
```

---

## Article Content Language

Article content is **not** translated at render time. The pipeline pre-rewrites clusters for each `(style, language)` variant. When serving the feed or an article:

1. `profile_service.get_reading_variant(profile, config)` returns `(style, language)` from the user's profile.
2. `article_service.get_feed()` and `get_expanded_cluster()` load rewrites for that variant from `cluster_rewrites`.
3. If the user's variant is missing, the service falls back to `default_style` / `default_language` from config.

## Updating Translations

When adding or changing translatable strings:

```bash
# Extract strings from templates and Python
pybabel extract -F babel.cfg -o messages.pot .

# Update existing catalogs with new/removed strings
pybabel update -i messages.pot -d translations

# Edit translations/ca/LC_MESSAGES/messages.po (and es, en)
# Fill in msgstr for each msgid

# Compile to .mo (required for runtime)
pybabel compile -d translations
```

To add a new language (e.g. French):

```bash
pybabel init -i messages.pot -d translations -l fr
# Edit translations/fr/LC_MESSAGES/messages.po
pybabel compile -d translations
```

Then add the new locale to `get_locale()`'s `best_match` list and to `config/app.yaml` under `rewriting.languages`.

## Scope

- **In scope:** User-facing UI (auth, setup, settings, feed, article, archive). Error messages. TTS labels. Relative timestamps. Article content (via pipeline).
- **Out of scope:** Admin dashboard (operator-facing). Email notifications (not yet implemented).
